# Phase 2.1: Shooting Feedback & Gunfeel Pass - Context

**Gathered:** 2026-03-09
**Status:** Ready for execution

<domain>
## Phase Boundary

本阶段只处理枪械射击链路的 shooter-side feedback：让玩家在开火后能清楚分辨 `player_hit`、`kill`、`environment_hit` 与 `miss`，并为 `pistol`、`smg`、`shotgun` 建立基础反馈个性。

实现范围限定在现有单文件 `index.html` 内，不引入模块拆分、外部音频资源或新的测试框架；重点是把当前 `attemptFire()` 的副作用堆叠拆成更清晰的本地 helper，同时保持 `duel` 与 `deathmatch` 都可读。

本阶段不扩展 grenade、barrel、melee 的统一反馈体系，也不做完整的 combat redesign；更激进的数值微调和难度联动留给后续 `Phase 3: Balance Pass`。

</domain>

<decisions>
## Implementation Decisions

### Experience direction
- 锁定 `Readable arcade`：先强调命中可信度和读场稳定，再追求爽感。
- 不做强全屏震动或持续遮挡准星的中心特效。

### Structural scope
- 锁定 `Local helpers`：继续在 `index.html` 内实现。
- 必须拆出 `shot result`、`feedback router`、`weapon feedback tuning` 这三个本地边界。
- 禁止继续把新反馈逻辑直接堆进 `attemptFire()` 的主分支里。

### Weapon identity scope
- 锁定 `Baseline identity`：本阶段只建立三把枪的基础差异与统一反馈框架。
- `Phase 3` 负责进一步细调强度、平衡和难度联动。

### Feedback priority
- 固定优先级为 `kill > player_hit > environment_hit > miss`。
- `near-miss` 只能消费真实 `miss`。
- `kill confirm` 必须早于或至少不被 `killcam` 吞掉。

### Claude's Discretion
- HUD 中心确认层的具体视觉形态可由 Claude 决定，但必须短促、可信、不遮挡目标。
- 命中确认的具体节流阈值可微调，但 `smg` 在 `deathmatch` 下必须比 `duel` 更克制。

</decisions>

<specifics>
## Specific Ideas

- 现有项目已经有 `tracer`、`muzzle`、`hitFlash`、`shake`、`near-miss`、`killcam`，所以本阶段不缺“反馈零件”，缺的是统一结果模型和事件优先级。
- 当前最危险的回归点不是“特效不够多”，而是：
  - 命中人与命中掩体的确认过于相似
  - `smg` 高频命中造成 HUD 噪音
  - `killcam`、`near-miss` 抢占真正命中确认

</specifics>

<code_context>
## Existing Code Insights

### Reusable Assets
- `attemptFire()`：当前枪械判定和副作用入口，适合作为 `shot result seam` 的插入点。
- `applyDamage()` / `handleEntityDeath()`：现有 damage 与 kill 提升入口，可用于 outcome promotion。
- `spawnTracer()`、`triggerMuzzle()`、`updateCameraFx()`、`updateFxTimers()`：可继续作为 secondary feedback layer。
- `state.gameMode`、`getDifficultyForMode()`、`getModeDifficultyPreset()`：可复用作 mode-aware tuning 输入。

### Established Patterns
- 当前项目偏向配置驱动与 guard clause。
- HUD 与 FX 已经通过全局 `fx`、`world` 对象维护短生命周期状态。
- DOM 结构是固定 overlay + per-frame update，不适合引入复杂组件系统。

### Integration Points
- HUD overlays: `#hitFlash`、`#nearMissFlash`、`#killCamText`、`#popups`
- Combat loop: `attemptFire()` → `applyDamage()` / `applyCoverDamage()` / `applyBarrelDamage()`
- Match flow: `endRound()` / `triggerKillCam()` / `startMatch()` / `setMode()`

</code_context>
